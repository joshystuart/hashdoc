# Coding rules
- Never add code comments. If the code is not clear enough on its own, then we need to simplify the code.

# Developing
- Never assume that what you have built works, always verify it.
- To verify UI, use the chrome devtools mcp to view the changes.
- To verify server/mcp, start the service and hit the relevant endpoints.
- Always run tests after changes to ensure they pass.
